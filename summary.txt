Course Master Microservices with Spring Boot, Docker, and Kubernetes.
1. Introduction to Microservices: Establishes the relevance of microservices, highlighting the transition from monolithic
applications to microservices architecture due to benefits like scalability and faster development.

2. Building Microservices with Spring Boot: Focus on best practices for creating REST APIs, including documentation
and data validation.
2.1 Open Api and Swagger

3. Containerization with Docker: Explains the essential role of Docker in containerizing applications for microservice
development.
Use of Dockerfile or Google Jib to build the docker image
Use of docker-compose to run multiple containers, setup the network,

4. Configuration Management: Introduces Spring Cloud Config Server for effective management of application configurations.
4.1 Spring Cloud Config Server is a centralized configuration server that provides centralized configuration management for
microservices by connecting to a source that can a db, a remote repository, or the local file system.
4.2 Refresh the configuration using the spring cloud bus that creates an event broadcasting channel to communicates
with AMQP brokers, such as RabbitMQ, and Kafka
4.3 Set up a WebHook in GitHub: Set up a webhook to automatically send a POST request to Config Service

7. Service Discovery and Registration: Discusses the implementation of Eureka Server for service discovery alongside
Spring Cloud Gateway for routing.
7.1 Spring Cloud Netflix's Eureka will act as a service discovery agent
7.2 Spring Cloud Load Balancer library for client-side load balancing
7.3 Spring Cloud Gateway will act as the entry point for routing requests to the appropriate microservice &
acts as a central Policy Enforcement Point (PEP):
* Routing (Both Static & Dynamic) Configuration made in the Gateway server like filters, predicates, etc
* Security (Authentication &Authorization)
* Logging, Auditing and Metrics collection

8. Resilience and Fault Tolerance: Covers resilient design patterns employing Resilience4j to enhance system robustness.

9. Observability and Monitoring: Teaches how to implement monitoring tools like Grafana and Prometheus for maintaining
system health.

10. Security: Focuses on securing microservices using frameworks like OAuth2, Keycloak, and Spring Security.

11. Event-Driven Microservices: Explores building asynchronous microservices using messaging systems like RabbitMQ and Kafka.

12. Deployment with Kubernetes: Guides on deploying microservices within a Kubernetes cluster and managing deployments using Helm.

13. Cloud Deployment: Covers the creation of Kubernetes clusters in cloud environments for efficient deployment.

14. Include the dependency manager project or BOM to have a central place to define the versions of the dependencies.

15. Create a projects Modules or libraries with common code to share between the microservices.
