Section 16 Helm
 Helm is a package manager for Kubernetes. It helps to automate the installation, rollback, upgrade of the
multiple K8s manifests with a single command.
 A chart is a collection of files that describe a related set of Kubernetes resources.
 A single Helm chart might b e used to deploy a simple app o r something complex like a
full web app stack with HTTP servers, databases, caches, and so on.
 A chart can have child charts and dependent charts as well. This means that Helm can install whole dependency tree of
a project with just a single command.

When a chart is installed all the kubernetes manifest files are loaded to launch the application. a chart is like a
package which contains all the resource dependencies to run an app or tool.

7. Create a helm chart for each profile: prod, dev, qa
$ mkdir prod-env # create a folder for prod env
$ helm create prod-env
Update the Chart.yaml file to have specific property values for each profile.
Example:
name: prod-env
appVersion: "1.0.0"
# add the chart dependencies.
* Update the values.yaml file to add the values for the prod-env chart and the other profiles.
These values are on the docker-compose.yml file.
Example of the prod values.yaml file:

The values from the prod-env/values.yaml file are used for example in the eazybank-common/templates/configmap.yaml,

11. Install Prometheus chart so it can get all the metrics from the MS
$ helm install kube-prometheus kube-prometheus
Check the values.yaml file:
additionalScrapings:
 enabled: true
 type: internal  # type internal to scrap metrics internally in the cluster
 jobList: [ {} ] # add the MS endpoints to scrape = scrape_configs
scrape_configs: # see kube-prometheus/configmap.yaml
 job_name: 'accounts'
         metrics_path: '/actuator/prometheus'
         static_configs:
           - targets: ['accounts.default.svc.cluster.local:8080']

Check the status of the targets:
http://localhost:9090/targets

12. Install Grafana Loki & Tempo in Kubernetes Cluster using Helm Charts
12.1 Install Grafana Loki
Loki and Tempo are components of Grafana.
Loki is responsible to aggregate all the logs that are generated by the individual microservices.
Tempo stores and lets you query TRACES —the end-to-end record of a single request as it flows through multiple services.
$ helm install loki grafana-loki # loki is the name of the release locally, grafana-loki is the folder and the name of the chart
$ helm status loki

On Grafana Tempo check the configuration for otlp (Open Telemetry config) in values.yaml file:
distributor:
        receivers:
          otlp:
            protocols:
              grpc:
                endpoint: 0.0.0.0:4317
              http:
                endpoint: 0.0.0.0:4318

12.1 Install Grafana Tempo
$ cd helm
$ helm install tempo grafana-tempo

The MS open telemtry will send the traces at this url:
otelExporterEndPoint: http://tempo.default.svc.cluster.local:4318

with this command you can get the services available in the cluster to get the ip and tcp port:
default local url is tempo.default.svc.cluster.local for the default namespace.
kubectl get services

12.2 Install Grafana Alloy
Grafana Alloy agent reads pod/container logs from Kubernetes and pushes them to Loki.
$ cd helm
$ helm install alloy grafana-alloy
$ helm status alloy

check logs from any kubernetes pod:
$ kubectl get pods # find the pod name
$ kubectl logs <pod_name>

15. How to make changes on the env var of the MS or the configs in K8s like increasing the # of replicas.
Use Helm to make changes on the MS and K8s settings.
e.g of modification got to eazybank-services/gatewayserver/values.yaml file:
 * update the docker tag to S11 which doesn't have security set
 * increase the gateway's number of replicas.
$ helm status eazybank
$ cd prod-env
$ helm dependencies build
$ cd .. # go back to the root folder environment/ where the prod-env/ is located
$ helm upgrade eazybank prod-env # eazybank is the release name and prod-env where the chart is. the revision number will be increased

16. How to rollback changes using Helm
it is going to rollback your entire Kubernetes cluster components to the previous desired state.
$ helm history eazybank
$ helm rollback eazybank <revision_number>
$ helm rollback eazybank 1
* check the pod's event to see how the gateway server is rolled back to previous docker tag
$ helm history eazybank # see history of the deployments for a particular release

17. How to uninstall all the MS using a Helm command
$ helm uninstall <release_name> # uninstall the release
$ helm ls # list all the releases. The NAME column is the release name
$ helm uninstall <release_name>
$ helm uninstall eazybank # uninstall the release with all the MS
* Make sure to delete the Grafana's Persistent Volume Claims (PVC) manually to avoid issue when reinstalling the helm chart again.
